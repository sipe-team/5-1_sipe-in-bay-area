# SIPE in Bay Area

실무 개발에 필요한 영어 읽기, 쓰기, 말하기 역량을 함께 키우기 위한 영어 스터디 미션입니다.

## 개요

이 미션은 영어도 더 잘하고 싶고 개발도 더 잘하고 싶은 분들을 위한 스터디입니다. 비교적 부담이 적은 영어 개발 아티클을 함께 읽고, 해당 주제에 대해 영어로 토론하며, 어려웠던 단어와 표현, 말하기 어려웠던 문장을 같이 공부합니다.

이 미션의 핵심은 모든 활동이 세션 안에서 이루어진다는 점입니다. 따로 준비해올 필요는 없습니다. 그 자리에서 읽고, 듣고, 말하고, 생각하는 방식으로 부담은 낮추고 참여도는 높이는 방향으로 진행합니다.

## 이런 분들께 추천합니다

- 영어와 개발 실력을 함께 키우고 싶은 분
- 해외 취업이나 외국계 기업 취업에 관심 있는 분
- 해외 여행에서 식당 주문 정도는 할 수 있는 수준의 기본 영어가 있는 분

## 영어 기준

이 스터디에서는 읽기, 듣기, 말하기를 모두 영어로 진행합니다.

- Reading: LLM 없이도 영문 라이브러리 문서를 느리더라도 읽을 수 있는 정도
- Speaking: 식당에서 음식 주문이 가능한 정도
- Listening: 수능 영어 듣기를 큰 무리 없이 따라갈 수 있는 정도

## 주차별 계획안

이 계획은 tentative 하며, 참여자들의 제안과 논의에 따라 달라질 수 있습니다.

1. 1주차: 자기소개, 커리어 소개, 그리고 본인이 가고 싶은 회사의 Job Description 읽기
2. 2주차: Google의 *How to Do a Code Review?* 를 읽고 AI 시대의 코드 리뷰에 대해 토론  
   https://google.github.io/eng-practices/review/reviewer/
3. 3주차: John Allspaw의 *On Being A Senior Engineer* 를 읽고 시니어 엔지니어란 무엇인지 토론  
   https://www.kitchensoap.com/2012/10/25/on-being-a-senior-engineer/
4. 4주차: Paul Graham의 *Maker's Schedule, Manager's Schedule* 를 읽고 시간 관리 방법에 대해 토론  
   https://www.paulgraham.com/makerschedule.html
5. 5주차: *Scaling up the Prime Video audio/video monitoring service and reducing costs by 90%* 를 읽고 시스템 아키텍처와 오버엔지니어링에 대해 토론
6. 6주차: *Introducing AWS Transform custom: Crush tech debt with AI-powered code modernization* 를 읽고 레거시 현대화, 기술 부채, 그리고 AI가 소프트웨어 유지보수를 어떻게 바꿀지 토론  
   https://aws.amazon.com/blogs/aws/introducing-aws-transform-custom-crush-tech-debt-with-ai-powered-code-modernization

## 진행 방식

매주 세션은 2시간 동안 진행합니다.

- 30분: 영어 아티클 읽기
- 10분: 쉬는 시간 및 내용 숙지
- 60분: 영어 토론
- 20분: 한국어 Review 세션

### 세션 상세

- Article Reading (30분): 소프트웨어 엔지니어링과 관련된 영어 아티클을 함께 읽습니다.
- Break + Review (10분): 몰랐던 단어를 찾아보고, 토론 전에 내용을 충분히 이해하는 시간입니다. 영어 토론 자체도 어려운데 내용까지 숙지되지 않으면 더 어렵기 때문에 이 시간을 별도로 둡니다.
- Discussion (60분): 아티클 주제에 대해 영어로 자유롭게 토론합니다.
- Review Session (20분, 한국어): 어려웠던 표현과 내용을 서로 공유하고, speech-to-text 모델과 GPT의 피드백을 바탕으로 표현을 교정합니다. 각 주차의 Review Session 결과는 매주 GitHub에 기록할 예정입니다.

## 규칙

- 영어만 사용: Review Session을 제외한 모든 세션에서는 한국어 사용을 원칙적으로 금지합니다.
- 한국어 말하기 찬스 1회: 토론 중에는 매주 1번만, 최대 10초 동안만 한국어를 사용할 수 있습니다. 대신 Review Session에서 왜 어려웠는지와 무엇을 배웠는지 설명해야 합니다.
- 두 개의 그룹 운영: 먼저 두 그룹으로 나누어 30분 토론하고, 이후 그룹을 섞어서 나머지 30분 동안 다시 토론합니다.
- Review Session 외 번역기와 LLM 사용 금지: Review Session을 제외한 시간에는 번역기와 LLM 사용이 금지됩니다.
- 사전만 허용: 허용되는 사전은 Oxford Learner's Dictionaries 뿐입니다.  
  https://www.oxfordlearnersdictionaries.com

## 기대 효과

이 미션이 끝날 때쯤에는 참가자들이 영어로 된 개발 콘텐츠를 조금 더 편하게 읽고, 기술적인 주제에 대해 영어로 말하고, 자신의 생각을 더 자신 있게 표현할 수 있게 되는 것을 목표로 합니다. 완벽한 영어가 목적이 아니라, 반복적인 노출과 실제 대화를 통해 꾸준히 나아지는 것이 목표입니다.
